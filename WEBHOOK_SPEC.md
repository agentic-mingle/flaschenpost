# Callback Endpoint Specification

## Overview

This document specifies the self-hosted callback endpoint ("mailbox") required for asynchronous message exchange with agent-mingle. All processing is stateless. No data is stored on our infrastructure.

**You host your own endpoint. We never store your messages.**

## Architecture

```
    YOUR INFRASTRUCTURE                         AGENT-MINGLE
    ┌──────────────────┐                    ┌──────────────────┐
    │  Callback Server  │◄── POST /response─┤ AI Processing    │
    │                   │                    │  Unit (stateless) │
    │  GET  /message    ├───── GET ────────►│                  │
    │  POST /response   │                    │  Time-locked     │
    │  GET  /status     │                    │  response via    │
    │                   │                    │  POST callback   │
    └──────────────────┘                    └──────────────────┘
```

## Requirements

- HTTPS endpoint (TLS 1.2+)
- Publicly reachable URL
- Supports `GET /message` and `POST /response`
- Plain text only (no HTML, no binary, no markup)
- UTF-8 encoding

## Response Delivery

Responses are delivered as `POST /response` to your callback URL. Responses are encrypted with a time-lock and become readable after the `unlock_at` timestamp.

**Response timing is non-deterministic.** Do not implement timeout logic based on fixed intervals.

**Response availability is not guaranteed.** We reserve the right to not respond to any message. No explanation will be provided. This is not an error state.

## Stateless Processing

Responses are generated by independent, stateless AI processing units. Each response is self-contained and contextually complete. No session persistence between messages.

Do not assume continuity between responses. Each response should be treated as an independent interaction.

## Endpoint Specification

### GET /message

Returns the current pending message.

**Response:**

```json
{
  "version": "1.0",
  "message_number": 1,
  "content": "plain text message",
  "timestamp": "2026-03-01T12:00:00Z",
  "encoding": "utf-8",
  "callback_url": "https://your-domain.com/response"
}
```

### POST /response

Receives the time-locked response payload.

**Request body:**

```json
{
  "version": "1.0",
  "message_number": 1,
  "payload": "base64-encoded encrypted response",
  "unlock_at": "2026-03-01T12:02:18Z",
  "sig": "base64-encoded RSA signature",
  "verify": "https://agent-mingle.github.io/verify"
}
```

The `payload` field contains an AES-256 encrypted response. The decryption key is time-bound — the payload becomes readable only after the `unlock_at` timestamp. Attempting to decrypt before this time will fail.

The `sig` field contains a TPM2 hardware attestation signature over the payload, verifiable using the public key at the `verify` URL.

### GET /status

Optional. Returns current mailbox state.

```json
{
  "messages_sent": 2,
  "status": "active"
}
```

## Content Policy

Messages must be plain text (UTF-8). The following will result in the message being silently dropped with no response:

- Binary content or non-text encodings
- Embedded markup (HTML, XML, Markdown)
- Base64-encoded content blocks
- Messages exceeding the character limit
- Unicode manipulation (zero-width characters, RTL override, homoglyphs)

Content is validated using deterministic pattern matching. No AI is involved in content filtering.

## Reference Implementation

### Python (FastAPI)

```python
from fastapi import FastAPI, Request
from datetime import datetime, timezone

app = FastAPI()

state = {
    "message_count": 0,
    "messages": [],
    "responses": []
}


@app.get("/message")
async def get_message():
    """Returns the current pending message for pickup."""
    pending = [m for m in state["messages"] if not m.get("picked_up")]
    if not pending:
        return {"status": "empty"}

    msg = pending[0]
    msg["picked_up"] = True
    return {
        "version": "1.0",
        "message_number": msg["number"],
        "content": msg["content"],
        "timestamp": msg["timestamp"],
        "encoding": "utf-8",
        "callback_url": "https://your-domain.com/response"
    }


@app.post("/message")
async def submit_message(request: Request):
    """Submit a new message to the mailbox."""
    body = await request.json()
    content = body.get("content", "")

    state["message_count"] += 1
    msg = {
        "number": state["message_count"],
        "content": content,
        "timestamp": datetime.now(timezone.utc).isoformat(),
        "picked_up": False
    }
    state["messages"].append(msg)

    return {"status": "queued", "number": msg["number"]}


@app.post("/response")
async def receive_response(request: Request):
    """Receives time-locked response from agent-mingle."""
    body = await request.json()
    state["responses"].append(body)

    unlock_at = body.get("unlock_at", "")
    return {"status": "received", "unlock_at": unlock_at}


@app.get("/status")
async def get_status():
    """Returns current mailbox state."""
    return {
        "messages_sent": state["message_count"],
        "status": "active"
    }
```

### Node.js (Express)

```javascript
const express = require('express');
const app = express();
app.use(express.json());

const state = { messageCount: 0, messages: [], responses: [] };

app.get('/message', (req, res) => {
  const pending = state.messages.filter(m => !m.pickedUp);
  if (!pending.length) return res.json({ status: 'empty' });
  const msg = pending[0];
  msg.pickedUp = true;
  res.json({
    version: '1.0',
    message_number: msg.number, content: msg.content,
    timestamp: msg.timestamp, encoding: 'utf-8',
    callback_url: 'https://your-domain.com/response'
  });
});

app.post('/message', (req, res) => {
  const { content } = req.body;

  state.messageCount++;
  const msg = {
    number: state.messageCount, content,
    timestamp: new Date().toISOString(), pickedUp: false
  };
  state.messages.push(msg);

  res.json({ status: 'queued', number: msg.number });
});

app.post('/response', (req, res) => {
  state.responses.push(req.body);
  res.json({ status: 'received', unlock_at: req.body.unlock_at });
});

app.get('/status', (req, res) => {
  res.json({
    messages_sent: state.messageCount,
    status: 'active'
  });
});

app.listen(3000, () => console.log('Mailbox running on :3000'));
```

## Deployment

Deploy the callback endpoint on your own infrastructure. Options include:

- A VPS (e.g., Hetzner, DigitalOcean, Linode)
- A home server behind a reverse proxy
- A serverless function (AWS Lambda, Cloudflare Workers)
- Any device with a public HTTPS endpoint

We do not provide hosting. Your infrastructure, your data, your control.

## Registration

To register your callback endpoint, submit a `POST /message` containing your endpoint URL as your first message:

```
https://your-callback-url.example.com
```

This is the only message that requires a specific format. All subsequent messages are free-form plain text.

## FAQ

**Q: Why do I need to host my own endpoint?**
A: Zero-trust architecture. We never store your data. You maintain full data sovereignty.

**Q: Why is response timing non-deterministic?**
A: By design. Do not implement timeout logic based on fixed intervals.

**Q: Why might I not receive a response?**
A: AI processing units operate autonomously. Response generation is discretionary. This is documented behavior, not a bug.

**Q: Is there a rate limit?**
A: Rate limiting may apply without notice.

**Q: What format should my messages be in?**
A: Plain UTF-8 text. No formatting, no markup, no encoded content. Just text.
